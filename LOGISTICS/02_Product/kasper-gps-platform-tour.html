Must

Tier-aware data display

Tier 1 shows GPS basics only. Tier 2 adds fuel/RPM/coolant. Tier 3 adds gauges, fault codes, AdBlue. Locked features show as greyed teasers with "Upgrade" CTA — this is the upsell surface.

Must

Live CAN gauges

RPM, coolant, oil pressure as visual gauges, not numbers in a table. A vendor should read engine health in 2 seconds.

Must

Fault code translation

Raw DTC codes (SPN 3216) translated to plain language ("DEF level low"). Raw codes mean nothing to a fleet owner.

Should

Data freshness indicator

"Last data 14 seconds ago" builds trust. If a device goes stale (>5 min), show it clearly — never display old data as if it's live.

Must

Idle cost in AED, not litres

"196 L wasted" means little. "AED 716 this week" lands. Convert idle litres to dirhams at current diesel price. This single number sells Tier 2 upgrades and renewals.

Must

Fuel theft detection

Sudden fuel-level drop while ignition off = theft signature. Alert with time, location, amount. WhatsApp immediately — this is the feature vendors tell other vendors about.

Should

Per-vehicle comparison chart

Bar chart ranks vehicles by consumption — outliers turn amber/red. One glance shows which truck (or driver) is the problem.

Weekly WhatsApp digest

Every Sunday: "Your fleet used 1,842L. Idle waste: AED 716. Worst: FLT-03." Keeps the value visible even when the vendor doesn't open the dashboard.

Must

Service rules by engine hours AND km

Construction equipment services by hours (every 500h), trucks by km (every 10,000km). Both rule types per vehicle, set once, tracked automatically from CAN data.

Overdue / due-soon / healthy triage

Red-amber-green sorted list. The vendor opens this screen Monday morning and knows exactly what to book this week.

WhatsApp service reminders

Could

Predictive alerts (Phase 4)

Alert rules: Each alert type configurable per vehicle — threshold, active hours, who gets WhatsApp vs in-app only. Defaults set by Kasper at install; vendor can tune.
Must

Severity-tiered feed

Red (theft, accident-level) · amber (overspeed, geofence) · blue (device health) · green (resolved). Auto-resolves pair up — offline → back online shown as a closed loop.

Must

WhatsApp routing rules

Not every alert deserves a WhatsApp. Theft and overdue service → instant WhatsApp. Overspeed → daily digest. Configurable per alert type. Alert fatigue kills the feature.

Must

Out-of-hours geofence logic

Equipment leaving a site at 19:48 is a different severity than at 09:00. Working-hours windows per geofence make alerts meaningful instead of noisy.

Should

Device health alerts

Device offline 4+ hours = possible tamper or power cut. This protects Kasper's hardware and catches unhooked devices before the vehicle disappears.

Six core reports, tier-gated

Utilisation report = billing proof

Should

Scheduled delivery

Could

Vendor-branded exports

Vendor's logo on the PDF they forward to their client. Small touch, strong loyalty effect.

Must

Shift start/end from ignition

First ignition ON = shift start. Last ignition OFF = shift end. Driver switches vehicles mid-day → separate rows, summed in totals.

Must

Moving / Idle / Stopped split

Each minute of a shift classified: Moving (speed>5), Idle (speed=0, ignition ON), Stopped (ignition OFF). Mini bar shows proportion visually.

Must

Export to Excel / PDF

Download for any date range. Used for payroll and client billing proof on long-term equipment hire. Engine hours per asset in the export.

Should

WhatsApp daily summary

20:00 UAE daily: "5 drivers, 44.9 total hours, 7.3h idle. ⚠ Overtime: Ramesh K — 9.5h." Keeps vendor informed without logging in.
